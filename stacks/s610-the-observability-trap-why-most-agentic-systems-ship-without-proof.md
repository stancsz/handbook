# S-610 · The Observability Trap: Why Most Agentic Systems Ship Without Proof

Teams instrument their agentic systems obsessively — trace every LLM call, log every tool result, capture every token. But when it comes time to answer "does it actually work?" they switch to "vibe testing." The result is systems that look production-ready but have no real performance signal. The gap between observability and evaluation is where agentic quality goes to die.

## Forces

- **Observability is easy. Evaluation is hard.** Tracing framework calls takes an afternoon. Building a statistically rigorous eval harness takes weeks and requires domain expertise most teams don't have.
- **"Vibe LLM Judges" are a false floor.** Asking an LLM to rate another LLM's output 0–10 is not an eval — it's a popularity contest between two stochastic systems. It masks regressions and confirms the happy path.
- **The agent eval gap is structural.** Agents reason non-deterministically across variable-length loops. Classical unit tests can't cover it. A/B tests need volume. Regression tests need ground truth. Most teams have none of these.
- **Evals compound in value the more agents you add.** Single-agent outputs are auditable. Four-agent pipelines with async handoffs produce traces that are 200 steps long — no human can manually review them all. Without automated evals, you're flying blind.

## The move

**Build the eval harness before the agent, not after.**

### The minimum viable eval stack

- **Golden dataset:** 50–200 curated (input, expected_output, context) triplets covering happy paths, edge cases, and known failure modes. Curated by domain experts, not generated by the LLM being evaluated.
- **LLM-as-Judge with constraints:** If using an LLM judge, pair it with deterministic checks (regex, keyword presence, schema validation). Unconstrained LLM judges agree with themselves 70–80% of the time on borderline cases — enough to feel good, not enough to trust.
- **Step-level tracing:** Log every agent loop iteration as a structured event with (state_before, tool_call, tool_result, state_after). This turns the agent trace into a searchable, diffable artifact. LangSmith, Phoenix, or custom Redis + Grafana all work here.
- **Regression suite on every PR:** Treat agent evals like unit tests — they must pass before merging. "Works on my machine" is not acceptable when your machine is a probabilistic system.
- **Smoke test in staging with synthetic user behavior:** Inject the golden dataset into staging automatically, not manually. Catch regressions before they hit production traffic.

### The Shopify eval discipline (ICML 2025)

Shopify's Sidekick team (building a merchant-facing agentic assistant) learned the hard way that vibe testing doesn't scale:

> *"Vibe testing, or creating a 'Vibe LLM Judge' that's like 'Rate this 0-10', is not going to cut it. It needs to be principled and statistically rigorous, otherwise you should be shipping with a false sense of security."*

Their answer was GRPO-style (Group Relative Policy Optimization) training with structured LLM-based evaluation — moving from subjective scoring to statistically grounded signal. The goal: eval coverage that catches a 5% regression in merchant-facing responses before it ships.

### The observability/eval gap by the numbers

Of teams with agentic systems in production, **89% have observability** (trace logs, token tracking, latency metrics) but only **52% have evals**. The remaining 37% can see their agents run — they just can't prove they work.

## Gotchas

- **Observability without eval tells you the agent ran — not that it ran correctly.** A 200-step trace with zero evals is a very detailed way to watch a system fail silently.
- **LLM judges drift over time.** As the judge model updates, your eval threshold shifts. Pin model versions for eval harnesses, or your regression suite will have false positives from model drift, not agent regressions.
- **Golden datasets rot.** Product behavior, terminology, and user expectations change. An eval set that was accurate 6 months ago may be testing the wrong behavior. Budget time to refresh it quarterly.
- **Single-agent evals don't generalize to multi-agent.** Each agent-to-agent handoff introduces new failure modes (schema drift, timeout, context truncation). Evaluate at the workflow level, not just the agent level.
- **Cost observability without cost control is theater.** The Zylos research found enterprises average $85,521/month in AI operational costs — knowing you spent that is different from having circuit breakers that stop runaway loops before they hit five figures.
