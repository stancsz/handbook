# S-242 · Multi-Agent Team Architecture — Beyond the Single-Agent Plateau

A single agent handles 8 tasks capably. At 15, it starts missing steps, dropping context, and failing silently. The path forward is not a smarter prompt — it is a team.

## Forces

- **Single agents hit a competence ceiling on complex, cross-functional work.** When a task spans research, synthesis, and execution, a generalist agent bleeds context across steps and degrades. Comet ML measured a 73% performance drop when critical information sits in the middle of long contexts — the model hallucinates, personas bleed into each other, and guardrails buried under recent messages stop firing. — [Comet ML, Multi-Agent Systems Architecture, 2025](https://www.comet.com/site/blog/multi-agent-systems)
- **The orchestration model you choose shapes your entire failure surface.** LangGraph uses explicit state machines (high control, high boilerplate), CrewAI uses role-based abstraction (fast prototyping, less visibility), and AutoGen uses multi-turn conversation (natural for collaborative tasks, harder to debug). Choosing one means living with its failure modes. — [Gheware DevOps, LangGraph vs CrewAI vs AutoGen 2026](https://devops.gheware.com/blog/posts/langgraph-vs-crewai-vs-autogen-comparison-2026.html)
- **"Intelligence is table stakes — the unlock is everything around it."** After 2 billion agentic executions, CrewAI's core finding: most models are good enough. GPT-4o-mini runs significant production workloads. The companies that successfully reach production scale are not building smarter agents — they are solving Agent Operations: audit trails, human oversight, outcome trust, and error recovery. — [CrewAI Blog, Lessons From 2 Billion Agentic Workflows, Jan 2026](https://blog.crewai.com/lessons-from-2-billion-agentic-workflows)
- **The agent stack is stratifying into specialized layers.** Sandboxing (E2B, Modal, Firecracker wrappers), orchestration, tool registries, and observability are becoming distinct concerns with their own defensibility profiles. Going monolithic across these layers is a known anti-pattern. — [HN/Philipp Dubach, Don't Go Monolithic — The Agent Stack Is Stratifying, Jun 2025](https://philippdubach.com/posts/dont-go-monolithic-the-agent-stack-is-stratifying/)

## The move

Split work along role boundaries, give each agent a narrow mandate and its own memory, and orchestrate with the right abstraction level for your team's debugging tolerance.

- **Define agents by function, not capability.** Opensoul's marketing stack runs a Director, Strategist, Creative, Producer, Growth Marketer, and Analyst — each mapped to a real marketing agency role. Each agent owns its queue, executes autonomously on heartbeats, delegates to teammates, and reports progress. — [HN Show, Opensoul Open-Source Agentic Marketing Stack, Mar 2026](https://news.ycombinator.com/item?id=47336615)
- **Default to LangGraph for production, CrewAI for prototyping.** LangGraph's graph-based state machine gives you explicit control over transitions, retries, and checkpoints — worth the steeper learning curve. CrewAI ships working multi-agent teams fastest with its role-based model. If you start in CrewAI and hit a wall at 6-12 months, the rewrite to LangGraph is painful. — [Gheware DevOps, LangGraph vs CrewAI vs AutoGen 2026](https://devops.gheware.com/blog/posts/langgraph-vs-crewai-vs-autogen-comparison-2026.html)
- **Microsoft Agent Framework (AutoGen + Semantic Kernel merger) is the enterprise Azure path.** GA is planned for Q1 2026. If you are already in Azure, this convergence reduces framework sprawl. — [Gheware DevOps, LangGraph vs CrewAI vs AutoGen 2026](https://devops.gheware.com/blog/posts/langgraph-vs-crewai-vs-autogen-comparison-2026.html)
- **Use hierarchical coordination for directed tasks, peer coordination for exploratory ones.** Director/manager agents route work to specialists for high-throughput, predictable workflows. Peer-to-peer (agents communicating as equals) works better when the task structure is unknown upfront. — [Comet ML, Multi-Agent Systems Architecture](https://www.comet.com/site/blog/multi-agent-systems)
- **Add a guardrail layer between the planner and the tool layer.** Plan what to do in natural language; constrain how it executes in structured schemas. This separates the creative reasoning path from the safety-critical execution path and dramatically reduces hallucinated tool calls. — [Netguru, The AI Agent Tech Stack in 2025](https://www.netguru.com/blog/ai-agent-tech-stack)
- **Instrument before you scale.** LangSmith processes traces from 400+ companies. Arize Phoenix handles open-source evaluation. The most common production failure is deploying without a tracing backend — when the system misbehaves, you have no data to debug it. — [OptinAmpOut, Agent Observability Transforms Production AI, Oct 2025](https://www.optinampout.com/blogs/agent-observability-transforms-production-ai.html)
- **Treat iteration caps as a first-class design decision.** Agentic loops improve results only when they target a specific gap. Without a stop condition, agents loop indefinitely on vague goals — adding cost without quality. Set max iterations per task type. — [MarsDevs, Agentic RAG: The 2026 Production Guide](https://www.marsdevs.com/guides/agentic-rag-2026-guide)
- **Agentic RAG differs from standard RAG fundamentally.** Standard RAG: one-shot retrieve-then-generate. Agentic RAG: adaptive loop with planning, retrieving, evaluation, and stopping rules. Use agentic RAG for multi-step reasoning and cross-source synthesis; use standard RAG for scoped, grounded information search. — [Neo4j, What is Agentic RAG, May 2026](https://neo4j.com/blog/agentic-ai/what-is-agentic-rag/)

## Gotchas

- **Building the demo is the easy part.** Netguru's Omega assistant for sales found that integrating into daily workflows, handling edge cases, and maintaining usefulness over time were the hard problems — not the AI capability itself. — [Netguru, The AI Agent Tech Stack in 2025](https://www.netguru.com/blog/ai-agent-tech-stack)
- **Agent Ops takes 3-5x longer than agent logic.** From the 2-billion-execution report: companies underestimate the gap between "works in a notebook" and "runs reliably at scale with audit trails." The teams succeeding budget time accordingly. — [CrewAI Blog](https://blog.crewai.com/lessons-from-2-billion-agentic-workflows)
- **Tool calling accuracy drops sharply under production load.** GPT-4o achieves 85% tool calling accuracy in ideal conditions — that number degrades in long-running stateful sessions with noisy context. — [OptinAmpOut, Agent Observability, Oct 2025](https://www.optinampout.com/blogs/agent-observability-transforms-production-ai.html)
- **Migrating from single-agent to multi-agent is not additive.** Adding agents changes the communication patterns, failure modes, and observability requirements. Plan for it as a rewrite, not an extension. — [Gheware DevOps](https://devops.gheware.com/blog/posts/langgraph-vs-crewai-vs-autogen-comparison-2026.html)
